# Train-Consist-Management

The Train Consist Management App is a console-based Java application that simulates how a railway system manages a train’s consist, which is a collection of bogies attached to an engine.

## Use Cases Implemented

1. **UC1:** Build a Basic Goods Bogie (Classes & Objects)
2. **UC2:** Assemble a Complete Goods Train (Constructors & Object Arrays)
3. **UC3:** Standardize Bogie Interfaces (Interfaces & Polymorphism)
4. **UC4:** Dynamic Train Assembly (ArrayList)
5. **UC5:** Flexible Train Assembly (Method Overloading)
6. **UC6:** Store Fast-Lookup References (HashMap)
7. **UC7:** Sort Bogies by Capacity (Comparator)
8. **UC8:** Filter Passenger Bogies Using Streams (Stream API)
9. **UC9:** Group Bogies by Type (Collectors.groupingBy)
10. **UC10:** Count Total Seats in Train (reduce)
11. **UC11:** Validate Train ID & Cargo Codes (Regex)
12. **UC12:** Safety Compliance Check for Goods Bogies (Functional Interfaces)
13. **UC13:** Performance Comparison (Loops vs Streams)
14. **UC14:** Handle Invalid Bogie Capacity (Custom Exception)
15. **UC15:** Safe Cargo Assignment (try-catch-finally)
16. **UC16:** Sort Passenger Bogies by Capacity (Bubble Sort)
17. **UC17:** Sort Bogie Names (Arrays.sort)
18. **UC18:** Linear Search for Bogie ID (Array-Based Searching)
19. **UC19:** Binary Search for Bogie ID (Optimized Searching)
20. **UC20:** Exception Handling During Search Operations (Fail-Fast Validation)
